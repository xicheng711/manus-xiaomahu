App 核心逻辑与 UI 统一重构方案
你好！我们需要对“早上打卡”输入流程（image_0.png, image_2.png, image_3.png, image_4.png）和“护理简报”输出页面（image_1.png, image_5.png）进行一次基于用户反馈的核心优化和 UI 规范统一。

目标：更短的打卡路径、更智能的数据处理、逻辑严密的可视化简报、以及全应用统一的温馨马卡龙色调设计。

请严格按以下阶段执行：

阶段 1：设计规范与颜色统一 (Unified Design System)
在开始写代码前，请确定并统一全应用的使用颜色，风格定为：温暖、温馨、柔和的马卡龙色调，留白充足。

Primary Action (主操作): 统一使用柔和的湖蓝色 (#29C8B1 / #6EE7B7)。

Highlight (高亮/图表): 统一使用柔和的浅黄色 (#FBBF24 / #FDE68A)。

Warning/Notice (提醒/通知): 统一使用柔和的粉橙色 (#F472B6 / #FECDD3)。

Background (背景): 全屏使用柔和的淡色渐变，卡片使用带柔和阴影的白色圆角卡片。

按钮状态: 禁用状态统一改为淡淡的灰色，启用状态统一为上述主操作色。

阶段 2：后端与数据时序逻辑修正
修正简报数据源逻辑：早上打卡后生成的“简报简报”，其关于老人心情、吃饭情况、用药准时度的“昨天小结”部分，必须从数据库读取 昨日晚间打卡记录 (Yesterday Evening Check-in)。如果昨晚未打卡，该模块显示“未打卡”或“-”。

新增接口 getWeeklySleepDetailedData：需要一个接口获取过去 7 天的 sleep_segments JSON，用于前端计算总时长和苏醒次数，绘制趋势图。

阶段 3：重构早上打卡输入流程 (Input UI - 精简至 2 个主页面)
请修改打卡流程，确保用户体验极其顺畅。必须集成 Haptics (振动反馈)。

Remove Steps: 移除原流程中所有关于“照顾者心情”的提问。

Screen 1: 昨夜睡眠记录 (核心智能页面)

UI Required:

顶部标题：“早间打卡” | 进度条统一为 1/2。

问题卡片：保留人物昵称（如【11】昨晚睡得怎么样？）。

提供 Segmented Control (Radio buttons) 切换：【快捷记录】vs 【详细记录】。

【快捷记录】：保留 Slider 选择“一共睡了几个小时”。

【详细记录】：

列表展示时间段。点击“＋ 添加睡眠时间段”弹出 native time picker。用户添加多个段（如 22:00-02:00, 03:00-07:00）。

智能计算区（核心逻辑）：在用户添加完时段后，页面下方自动计算并显示以下文字：

“昨夜总睡眠时间：8 小时。夜间苏醒 1 次 (苏醒时间：02:00-03:00)。”

Colors: 启用“添加”按钮为 Primary 马卡龙湖蓝色，计算区文字使用浅黄色高亮重点。

Screen 2: 白天小睡 & 补充信息

问题 1（智能 Nap）：白天有小睡吗？ (Yes/No Toggle)。

如果点【Yes】，下方滑出一个 Segmented Control (单选卡片组)，选项为：0-30分钟, 30-1小时, 1小时-1.5小时。

问题 2（保留）：还有什么要补充的吗？ (Optional notes area)。

阶段 4：全新的“护理简报”可视化呈现 (Output UI - 时序与视觉修正)
当早上打卡完成，弹出简报页面 (参考图 image_1.png 与 image_5.png 的重构)。务必设计得美丽、温馨，使用圆角、线性图标。

引入图表库：确保已安装 react-native-gifted-charts。

简报页面排版设计 (从上到下 - 修正时序逻辑)

头部: AI 生成的极简“温暖的一句鼓励话”（不再显示今日建议，生成更快速）。分数可以保留。

昨日全天情况 (Yesterday Summary Badge Row)：

横向排列的 Icon 卡片（修正数据源为 YESTERDAY EVENING Check-in）：

昨日老人心情 | 昨日用药情况 | 昨日吃饭情况。

数据判空逻辑：如果昨晚未打卡，这部分卡片显示为灰色底，文字改为“未记录昨晚情况”。

昨夜睡眠可视化 (Last Night Sleep Visual - NEW Priority)：

聚焦昨夜数据。

如果使用详细记录模式（可视化甘特图）：使用一条水平时间轴（Timeline），清晰绘制出几点到几点在睡觉，中间哪里由于时段空缺是苏醒的。并在图下方用小字标出“计算得出昨夜总睡眠 8 小时，苏醒 1 次”。Accent color 使用浅黄色高亮睡眠块。

如果是快捷记录模式，用一个精美的环形图（Ring Chart）展示。

近一周睡眠趋势 (Weekly Sleep Trend)：

使用 <BarChart /> 展示过去 7 天。X 轴为日期。

柱子使用圆角马卡龙浅黄色。

智能点：在柱子下方或上方用小数字标出“夜间醒来次数”（同样基于详细记录的时间差计算）。

底部: 补充信息小结 + “保存长图分享到微信”的 Primary 湖蓝色圆角按钮 (保留现有 react-native-view-shot 逻辑)。
